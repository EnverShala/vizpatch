# Phase 2 Plan 03: Conversation History Live IMAP Summary

D-26 implementiert: Der Agent holt bei jedem Draft-Zyklus frisch aus IMAP (INBOX + Sent) den bisherigen Gesprächsverlauf und injiziert ihn in den Draft-Prompt via `{conversation_history}`-Placeholder.

## Was geändert wurde

### agent/src/imap_client.py
Zwei neue Methoden nach `append_to_drafts()`:

- **`fetch_thread_history(references, max_messages=6)`** — Sucht INBOX und Sent-Ordner nach Mails, die in `In-Reply-To` oder `References` eine der übergebenen Message-IDs enthalten. Nutzt `imap_tools.OR` + `imap_tools.query.H` für Header-SEARCH. Gibt deduplizierte, chronologisch sortierte Liste zurück (max `max_messages` neueste).

- **`fetch_sender_history(from_address, days=30, max_messages=6)`** — Absender-Fallback: sucht `FROM x` in INBOX und `TO x` in Sent für die letzten 30 Tage. Gleiche Dedup/Sort-Logik.

Neue Imports: `OR` aus `imap_tools`, `H` aus `imap_tools.query`, `timedelta` aus `datetime`.

### agent/src/generate.py
- **`_HISTORY_BODY_MAX_CHARS = 800`** — Konstante für Body-Truncation.
- **`_truncate_body(body, max_chars=800)`** — Kürzt Body auf max. 800 Zeichen, hängt `\n[... gekürzt ...]` an.
- **`_build_history_block(history)`** — Baut den `{conversation_history}`-Prompt-Block aus einer MailMessage-Liste. Separator `\n\n---\n\n` zwischen Einträgen.
- **`generate_draft_text()` Signatur** — neuer Keyword-Parameter `conversation_history: list | None = None` (Backward-kompatibel, Default None).
- **`prompt.format()` Aufruf** — neuer Key `"conversation_history": history_block`.

### agent/prompts/generate.txt
Neuer Abschnitt `# Bisheriger Gesprächsverlauf (wenn vorhanden)` mit `{conversation_history}`-Placeholder zwischen `# Firmen-Kontext` und `# Eingehende E-Mail`.

### agent/src/main.py
- **`_process_one()` Signatur** — neuer Parameter `imap: "ImapClient"` (String-Literal-Type).
- **History-Fetch-Block** im REPLY_NEEDED-Pfad (vor `generate.generate_draft_text()`): normalisiert `references`/`in-reply-to`-Header defensiv gegen tuple/str/list, ruft `fetch_thread_history` bei vorhandenen References, sonst `fetch_sender_history`.
- **`generate.generate_draft_text()`-Aufruf** — neues Keyword-Argument `conversation_history=conversation_history`.
- **`_poll_once()`** — ruft `_process_one(msg, config, anthropic_client, logger, imap)` mit `imap`-Instanz.

## Fehlerfälle die graceful degradiert werden

| Fehlerfall | Verhalten | Log-Event |
|------------|-----------|-----------|
| Sent-Ordner nicht erreichbar (`folder.set()` wirft) | Warning-Log, INBOX-Ergebnisse trotzdem genutzt, Draft wird gebaut | `history_folder_not_found` mit `extra={"folder": ...}` |
| IMAP SEARCH fehlschlägt (Timeout, Encoding-Fehler) | Warning-Log, Suche in aktuellem Folder übersprungen, andere Folder fortgesetzt | `history_search_failed` mit `extra={"folder": ...}` |
| `fetch_sender_history` Folder-Fehler | Warning-Log, fehlender Folder übersprungen | `history_fetch_failed` mit `extra={"folder": ...}` |
| Leere references-Liste | Sofortiger Early-Exit, keine IMAP-Calls | — |
| Kein references-Header | Fallback auf Absender-Suche (30 Tage) | — |

In allen Fällen wird der Draft ohne oder mit reduziertem Verlauf gebaut — keine Exception propagiert nach außen, kein Poll-Zyklus unterbrochen.

## DSGVO-Rationale (D-26)

Keine persistente Speicherung der History in State-DB oder Files. Der Agent liest nur, was im Postfach des Betreibers sowieso vorhanden ist. Löschen im Postfach = automatisches Vergessen beim nächsten History-Fetch. Datenminimierung (Art. 5) und Recht auf Löschung (Art. 17) erfüllt by design.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Test-Assertion für _truncate_body angepasst**
- **Found during:** Task 2 GREEN phase
- **Issue:** Plan spec `len(result) <= 815` und PATTERNS.md `assert len(result) <= 815` sind inkonsistent mit dem implementierten Suffix `\n[... gekürzt ...]` (18 Zeichen), der 800 + 18 = 818 Zeichen ergibt. Test schlug fehl.
- **Fix:** Test-Assertion auf `<= 820` angepasst (818 ist korrekt, 820 gibt einen kleinen Puffer). Implementierung unverändert.
- **Files modified:** agent/tests/test_generate_with_history.py

### Known Issues (Not Fixed)

**1. DeprecationWarning: `datetime.utcnow()` in `fetch_sender_history`**
- Python 3.14 hat `datetime.utcnow()` als deprecated markiert. Spec (02-PATTERNS.md) nutzt diese Funktion explizit. Die 2 Warnings im Test-Output sind nicht kritisch (Tests laufen grün). Fix (auf `datetime.now(timezone.utc)`) in einem Folge-Plan sinnvoll.
