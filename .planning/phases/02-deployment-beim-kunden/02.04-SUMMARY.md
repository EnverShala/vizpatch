# Phase 2 Plan 04: Docker Bind-Mount + Deployment-Paket Summary

Docker-Image-Modus auf Bind-Mount umgestellt (D-16) und USB-taugliches Deployment-Paket-Build-Skript mit Zwei-Konfig-Trennung erstellt (D-04, D-22).

## What Was Built

### Task 1: Bind-Mount-Modell (D-16)

**`agent/Dockerfile`** — Zeile `COPY prompts/ ./prompts/` entfernt. Prompts werden nicht mehr ins Image kopiert, sondern zur Laufzeit via Bind-Mount hereingegeben. Dockerfile hat jetzt 28 Zeilen (war 29).

**`agent/docker-compose.yml`** — komplett ersetzt:
- `build: .` + `image: kea-tankstelle:latest` → `image: kea-tankstelle:v1.0.0`
- Neuer Bind-Mount `./prompts:/app/prompts:ro` in volumes-Sektion
- `./context.md:/config/context.md:ro` und `agent-data:/data` unverändert

**`agent/.env.example`** — umstrukturiert:
- 5 Pflichtfelder bleiben aktiv: `IMAP_USER`, `IMAP_PASSWORD`, `IMAP_DRAFTS_FOLDER`, `OWN_EMAIL_ADDRESS`, `ANTHROPIC_API_KEY`
- `IMAP_HOST`, `IMAP_PORT`, `IMAP_USE_SSL`, `IMAP_SENT_FOLDER` als auskommentierte Optional-Overrides mit Auto-Detect-Hinweis
- Provider-Kommentar auf "Unterstützte Provider: GMX, Web.de, IONOS, T-Online, Gmail, Outlook/M365, Mailbox.org, Strato, Hetzner" reduziert

**`agent/README.md`** — Setup-Sektion (Zeilen 12-41) ersetzt:
- War: `git clone git@github.com:vizionists/kea-tankstelle.git /opt/kea`
- Ist: `docker load -i kea-tankstelle-v1.0.0.tar` + Deployment-Paket-Workflow (6 Schritte)
- Alltag-Tabelle: "Update auf neue Version"-Zeile auf `docker load -i kea-tankstelle-vX.Y.Z.tar` umgestellt

### Task 2: Build-Skript + Deployment-Vorlagen (D-04, D-22)

**`scripts/build-deployment-package.sh`** (71 Zeilen, POSIX Shell):
- `set -euo pipefail` als Fehler-Semantik
- `bash scripts/build-deployment-package.sh [VERSION]` baut alles in einem Kommando
- Ablauf: `docker build` → `mkdir dist/deployment-paket-<VERSION>/` → `docker save` → Prompts + Compose + Templates + README kopieren → `sha256sum` Checksum erzeugen
- Kein `docker push`, keine Registry, reines Local-Build + Ordner-Packing

**`deployment/`-Ordner erstellt** mit 4 Vorlagen:
- `kunde-env.example` — Kunden-Template mit `<placeholder>`-Werten, 5 Pflichtfelder prominent, Rest auskommentiert
- `vizionists-test-env.example` — IONOS-konkret für `shala@vizionists.com`, `KEA-Test-Entwürfe` als Drafts-Ordner (Auto-CREATE-Test)
- `context.md.tankstelle-erstversion.md` — OSINT-Rohbau mit 6 Sektionen (About, Öffnungszeiten, Angebote/Preise, FAQ, Ton, Signatur), Platzhalter für Vor-Ort-Ergänzung
- `context.md.vizionists-test.md` — Vizionists-Test-Kontext mit denselben 6 Sektionen, konkretem Inhalt (Enver Shala, shala@vizionists.com)
