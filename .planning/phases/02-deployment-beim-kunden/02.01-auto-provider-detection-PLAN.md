<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DNS-Resolver → Provider-DNS | MX-Lookup gegen öffentliche DNS-Server; nicht vertrauenswürdiger Rückgabekanal |
| .env → config.py | IMAP-Credentials überqueren Filesystem-Grenze |
| Env-Variablen → Logs | Fehlermeldungen dürfen keine Credentials in Klartext leaken |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.01-01 | Spoofing | MX-Lookup (Provider-Detection) | mitigate | MX-Muster erfordern Substring-Match gegen bekannte Hoster; kein blindes Host-Assignment aus MX. Bei Fehltreffer: RuntimeError statt Fallback auf Attacker-Host. |
| T-02.01-02 | Information Disclosure | RuntimeError-Text bei Auto-Detect-Failure | accept | Error-Text enthält nur Domain-Namen, keine Credentials — steht in Logs. Domain ist ohnehin öffentlich sichtbar. |
| T-02.01-03 | Denial of Service | DNS-Lookup blockt Startup | mitigate | `_get_mx_host()` fängt alle Exceptions (`NoAnswer, NXDOMAIN, NoNameservers, Exception`) — Startup bricht nicht mit Netzwerk-Timeout ab, fällt auf RuntimeError mit klarer .env-Anweisung. |
| T-02.01-SC | Tampering | dnspython Package | mitigate | RESEARCH.md § Package Legitimacy Audit: `dnspython 2.8.0` slopcheck [OK]. Kein Human-Checkpoint nötig, da nicht [ASSUMED]/[SUS]. |
</threat_model>
