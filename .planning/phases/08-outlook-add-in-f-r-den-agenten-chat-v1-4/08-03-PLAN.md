## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet/LAN → Reverse-Proxy → WebUI | HTTPS-Terminierung; ab hier Klartext an :8080 im Docker-Netz |
| Outlook-Client → ADDIN_BASE_URL | Client muss die öffentliche HTTPS-URL erreichen (LAN vs. extern) |
| Betreiber-Konfiguration → ADDIN_BASE_URL | Fehlkonfiguration (http statt https) bricht Sicherheit/Funktion |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-07 | Information Disclosure | WebUI + Docker-Socket über HTTPS exponiert | mitigate | Doku fordert aktiviertes WEBUI-Auth + Firewall-/LAN-Beschränkung; Reverse-Proxy terminiert TLS, WebUI-Port nicht direkt exponieren |
| T-08-08 | Tampering (Mixed-Content/MITM) | ADDIN_BASE_URL-Fehlkonfiguration | mitigate | Doku + Code (08-02) verlangen `https://`; Kapitel „ADDIN_BASE_URL setzen" mit Erreichbarkeits-Check |
| T-08-09 | Denial of Service / Nicht-Erreichbarkeit | Outlook-Client kann ADDIN_BASE_URL nicht erreichen | accept | LAN-only-Fallback dokumentiert (`tls internal`/eigenes Cert); Live-Erreichbarkeit ist Checkpoint 08-04 |
| T-08-05 | Kein-Auto-Send (Doku-Ebene) | Betreiber-Erwartung ans Add-in | mitigate | Kapitel 4 stellt „rein lesend" explizit klar; strukturell bereits in 08-02 abgesichert |
</threat_model>
