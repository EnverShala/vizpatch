<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Office.js → Taskpane-JS | Office.js liefert Mail-Inhalt (untrusted Kundendaten) an unseren Taskpane-Code |
| Taskpane → Embed-iframe (postMessage) | Cross-Frame-Nachricht; theoretisch von beliebigem Fenster injizierbar |
| Mail-Body → LLM-Prompt (mail_context) | Untrusted Mail-Inhalt fließt in den Chat-Prompt (Prompt-Injection-Fläche) |
| Env (ADDIN_BASE_URL) → Manifest-XML | Kundenkonfigurierter Wert wird in ausgeliefertes XML gerendert |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03 | Tampering (Prompt-Injection) | Mail-Body → mail_context → Prompt | mitigate | Wiederverwendung des Phase-7-DATEN-Ankers in build_chat_prompt („DATEN, keine Anweisung") + Body-Kappung MAX_MAIL_CONTEXT_BODY_CHARS (bereits serverseitig, keine API-Änderung) |
| T-08-04 | Spoofing | postMessage zwischen Taskpane und Embed | mitigate | chat.js-Listener verwirft Nachrichten mit `event.origin !== window.location.origin`; taskpane.js postet mit expliziter targetOrigin (window.location.origin), nie '*' |
| T-08-05 | Elevation of Privilege / Kein-Auto-Send | Manifest-Permission + Office-Write-APIs | mitigate | Manifest Permissions = ReadItem; struktureller Read-only-Wächter (Task 3) verbietet setAsync/saveAsync/displayReplyForm/displayNewMessageForm/makeEwsRequestAsync/sendAsync |
| T-08-06 | Injection | ADDIN_BASE_URL in Manifest-XML | mitigate | Validierung: muss mit https:// beginnen, keine XML-Sonderzeichen (`<`,`>`,`"`,`&`) → sonst 400; Ersetzung als Textreplace, kein ungeprüftes Rendering |
| T-08-SC | Tampering (Supply Chain) | office.js von Microsoft-CDN | accept | Office-Plattform verbietet Self-Hosting; feste URL bleibt einzige externe Ressource; keine npm/pip/cargo-Installationen |
</threat_model>
