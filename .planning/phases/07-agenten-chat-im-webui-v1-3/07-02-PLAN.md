## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| context.md/style.md/agent_status → System-Prompt | vom Betreiber/System befüllt; landet im LLM-Prompt = Prompt-Injection-Fläche |
| Browser → /chat/*/send | User-Message ist untrusted Input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Tampering | Prompt-Injection via context.md/style.md/Status | mitigate | Injection-Anker im chat-system.txt kennzeichnet diese Abschnitte explizit als DATEN, nicht Instruktionen (Muster context-seed.txt); Test prüft Anker-Präsenz |
| T-07-06 | Information Disclosure | Status-Fehlertext könnte Interna leaken | accept | Status ist ohnehin nur dem auth-geschützten Betreiber sichtbar; keine Secrets im agent_status.json (Kontrakt 05.02) |
| T-07-07 | Tampering | .format-KeyError / Prompt-Bruch durch {}-Zeichen in context.md | mitigate | Template-Füllung via .replace statt .format; Acceptance-Gate verbietet .format im Assembler |
| T-07-SC | Tampering | keine neuen Package-Installs | accept | keine neuen Dependencies |
</threat_model>
