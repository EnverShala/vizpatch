| Chat → LLM | Kosten-/Token-Grenze überschritten = direkte Kostenwirkung |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-08 | Denial of Service | Chat-Kosten/Spam | mitigate | Rate-Limit CHAT_RATE_LIMIT_PER_MIN (Default 20, D-60) via slowapi; max_tokens-Deckel; Verlaufs-Trunkierung auf CHAT_HISTORY_TOKEN_BUDGET; 429-Test |
| T-07-09 | Tampering | manipulierte history im Request (client-gehalten) | mitigate | serverseitige Trunkierung + Struktur-Validierung (nur role/content, sonst verworfen); Budget deckelt Prompt-Größe unabhängig vom Client |
| T-07-10 | Elevation of Privilege | Chat als Mail-Sende-Vektor (Scope-Creep) | mitigate | struktureller Guard-Test: chat.py hat keinen imap/smtp/draft-Pfad (D-63); Kein-Auto-Send-Hinweis auch im System-Prompt (07-02) |
