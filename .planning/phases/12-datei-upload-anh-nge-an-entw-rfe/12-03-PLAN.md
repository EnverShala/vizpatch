<objective>
Ergänzt das bestehende Chat-Partial (`webui/src/templates/_chat.html` + `webui/static/chat.js`) um ein
minimales Datei-Upload-Widget (HTMX/vanilla-JS, konsistent zum bestehenden Chat-UI, --skip-ui): Datei
wählen → an `POST /chat/{agent_id}/upload` streamen → zurückgegebene Metadaten dem nächsten Chat-Send als
`attachment_meta` anhängen (ATT-03). Abschließend ein Live-Abnahme-Checkpoint für den End-to-End-Fluss.

Purpose: Der Betreiber lädt ad-hoc eine Datei hoch und bittet den Agenten, sie an einen Entwurf zu hängen —
ohne parallelen UI-Pfad, nahtlos im bestehenden agentischen Chat.
Output: Upload-Widget + Metadaten-Anhang-Logik + Status-/Fehleranzeige + Live-Abnahme.

  <read_first>

  </read_first>
  <action>
    Füge in `_chat.html` in `#chat-form` ein `<input type="file" id="chat-file-input">` (kein `accept`-Filter —
    alle Dateitypen, D-90) plus einen dezenten Upload-Button/Label ein, konsistent zum bestehenden Formular-Stil.
    Ergänze in `chat.js` (innerhalb der IIFE) Handles für den Datei-Input und eine Modul-lokale Variable
    `pendingAttachment = null`. Bei Dateiauswahl: baue eine `FormData` mit `file` + `session_id` (die bestehende
    `sessionId`) und `fetch('/chat/' + agentId + '/upload', {method:'POST', body: fd})`. Bei `res.ok` das JSON
    parsen und `pendingAttachment = {dateiname, groesse, mimetyp}` merken; bei Fehler (413/andere) den Response-Text
    lesen und eine dezente Statuszeile im `#chat-log` anzeigen (eigene Klasse `chat-upload-status`, Muster
    `.chat-tool-activity`). Zeige auch bei Erfolg eine kurze Statuszeile („Anhang bereit: <dateiname>"). Ergänze in
    `chat.css` einen schlichten Stil für `.chat-upload-status` analog `.chat-tool-activity`. Kein Dateiinhalt wird
    je in `history`/`localStorage` gespeichert — nur der Server hält die Datei (Pending-Store).
  </action>
  <verify>
    <automated>cd webui && grep -q 'type="file"' src/templates/_chat.html && grep -q "/upload" static/chat.js && grep -q "chat-upload-status" static/chat.css && echo OK</automated>
  </verify>
  <acceptance_criteria>
    - `webui/src/templates/_chat.html` enthält `type="file"` innerhalb von `#chat-form`.
    - `webui/static/chat.js` enthält einen fetch auf `'/chat/' + agentId + '/upload'` und setzt `pendingAttachment` bei Erfolg.
    - Bei Fehlerantwort wird eine `chat-upload-status`-Statuszeile ins `#chat-log` geschrieben (kein stiller Fehlschlag).
    - `webui/static/chat.css` enthält eine Regel für `.chat-upload-status`.
    - Der grep-Verify-Befehl gibt `OK` aus (exit 0).
  </acceptance_criteria>
  <done>Datei-Input + Upload-fetch + Status-/Fehleranzeige vorhanden; Metadaten in pendingAttachment gemerkt.</done>
