<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM → Tool-Handler | Das LLM liefert Tool-Argumente (untrusted, potenziell prompt-injiziert aus Mail-Inhalt). session_id gehört NICHT dazu (serverseitig injiziert). |
| Tool-Handler → IMAP-Server | APPEND von RFC-5322-Bytes in den Drafts-Ordner (nie SMTP/Send). |
| Pending-Store → Dateisystem (tmp) | tmp-Upload-Datei wird gelesen und muss zuverlässig gelöscht werden. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Spoofing | entwurf_mit_anhang session_id | mitigate | session_id NICHT im input_schema — serverseitig via _SESSION_SCOPED_TOOLS injiziert; LLM kann kein fremdes Handle konstruieren (Muster mail_in_papierkorb, Pattern 1) |
| T-12-02 | Information Disclosure | Datei-Rohinhalt → LLM (D-96) | mitigate | entwurf_mit_anhang NICHT in _ANON_AWARE_TOOLS; Result trägt nur Metadaten; Metadaten-DATEN-Block (Task 3) enthält nie Bytes; Test `_result_never_contains_raw_content` erzwingt das |
| T-12-03 | Denial of Service | tmp-Datei-Leck bei Fehlversuchen | mitigate | tmp-Cleanup im finally in JEDEM Ausgang (Erfolg/IMAP-Fehler/kein-Pending, Pitfall 5); Pending-Store-TTL 3600s |
| T-12-04 | Elevation of Privilege | injizierter SMTP-/Send-Pfad im neuen Werkzeug | mitigate | AST-Wächter `_scan_ast_for_forbidden_smtp_send_api` + Schema-Wächter `_scan_tool_schemas_for_forbidden_send_patterns` decken entwurf_mit_anhang automatisch ab (D-95/ATT-04) |
| T-12-05 | Tampering | überlanger Anhang (Kosten/Provider-Limit) | mitigate | Defense-in-Depth-Größenprüfung `MAX_ATTACHMENT_MB` gegen Rohgröße im Handler (Primärprüfung im Upload-Endpoint, Plan 12-02) |
| T-12-SC | Tampering | Package-Installs | accept | Keine neuen Packages (12-RESEARCH Package Legitimacy Audit) — Gate n/a |
