nächsten Chat-Turn als referenzierbaren Anhang verfügbar machen — die eigentliche MIME-/APPEND-Arbeit
erledigt das Werkzeug aus 12-01.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Upload-Route | Untrusted Multipart-Body (Dateiname, Größe, Inhalt) überquert hier die Grenze; nur authentifizierte Session. |
| Browser → chat_send attachment_meta | Untrusted JSON-Metadaten aus dem Verlauf; nur der EXISTIERENDE Pending-Upload wird per session_id konsumiert. |
| Upload-Route → Dateisystem (tmp) | Streaming-Write, server-generierter Dateiname. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-06 | Denial of Service | große/viele Uploads (Disk/Memory) | mitigate | MAX_ATTACHMENT_MB Live-Byte-Zähler + Streaming über file.file.read (kein Full-Memory-Load) + tmp-Cleanup bei 413; @limiter.limit wie chat_send |
| T-12-07 | Tampering | Path-Traversal über UploadFile.filename (z.B. ../../etc/passwd) | mitigate | os.path.basename(file.filename) — Dateiname nur für Anzeige/Content-Disposition; tmp-File bekommt server-generierten Namen via tempfile.mkstemp (ASVS V5) |
| T-12-08 | Spoofing/Info Disclosure | unautorisierter Upload | mitigate | auth.require_setup + auth.require_auth (dieselbe Kombination wie chat_send, D-95, ASVS V2); agent_id-404-Guard + AGENT_ID_PATTERN (ASVS V4) |
| T-12-09 | Tampering | gefälschte attachment_meta im Verlauf (Fake-Upload) | mitigate | Metadaten registrieren nur der authentifizierte Upload-Endpoint serverseitig; das LLM/history kann keinen Pending-Upload anlegen, nur den EXISTIERENDEN per session_id konsumieren (Security Domain Prompt-Injection) |
| T-12-10 | Information Disclosure | Dateiinhalt über attachment_meta ans LLM (D-96) | mitigate | chat_send übergibt nur Metadaten-dict (dateiname/groesse/mimetyp), nie Bytes; DATEN-Block in 12-01 trägt keinen Inhalt |
| T-12-SC | Tampering | Package-Installs | accept | Keine neuen Packages (python-multipart bereits installiert; email/mimetypes/tempfile stdlib) — Gate n/a |
