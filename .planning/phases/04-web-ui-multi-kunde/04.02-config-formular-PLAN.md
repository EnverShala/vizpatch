## Phase Goal

**As a** Tankstellen-Betreiber ohne SSH-Kenntnisse, **I want to** im Browser ein Konfig-Formular ausfüllen und speichern, **so that** der Vizpatch meine IMAP-Zugangsdaten, meinen Drafts-Ordner-Namen und meinen `context.md` bekommt, ohne dass ich jemals ein Terminal öffnen muss.

<objective>
UI-01 (Basic-Auth + Konfig-Seite) + UI-02 (Formular liest/schreibt `.env` und `context.md` mit Masking + chmod 600) ende-zu-ende funktional. Nach diesem Plan hat die WebUI erstmals fachlichen Wert für den Betreiber.

Purpose: Ohne dieses Formular hat der Betreiber keinen Zugriff auf die Kernkonfig — SSH bleibt erforderlich. Mit diesem Slice ist die 30-Minuten-Erst-Konfig per Browser abbildbar (siehe ROADMAP.md Estimation Phase 4 "~30 Min Erst-Konfig per WebUI vor Ort").

Output: `auth.py`, `config_io.py`, `main.py` erweitert um `GET /` + `POST /save` + `GET /_auth_check`, zwei Jinja2-Templates, CSS-File, 3 pytest-Test-Files mit >=8 grünen Tests. `deployment/kunde-env.example` um WebUI-Env-Vars erweitert.
</objective>
