## Design-Entscheidung: webui ohne depends_on: agent (W8)

Der `webui`-Service hat bewusst KEIN `depends_on: agent` in docker-compose.yml. Grund: webui muss auch bei agent-Crash oder agent-Konfigurationsfehler erreichbar bleiben, damit der Betreiber via UI restarten, Config korrigieren oder Update einspielen kann. Ein `depends_on: agent` würde webui in einen unbrauchbaren Zustand bringen wenn agent nicht startet — genau die Situation, in der der Betreiber das UI am dringendsten braucht.

Diese Entscheidung ist NICHT Teil von D-27..D-44 aus CONTEXT.md, sondern Umsetzungs-Freiheit gemäß RESEARCH.md-`Claude's Discretion`-Sektion. Trade-off: Beide Services starten parallel auch bei erstem `up -d` — akzeptabel, da webui-Startup unabhängig vom agent-Status funktioniert (nur /healthz + Formular funktionieren ohne agent; Status-Kachel zeigt "not_created" bei fehlendem agent-Container, was die korrekte UX ist).
</threat_model>
