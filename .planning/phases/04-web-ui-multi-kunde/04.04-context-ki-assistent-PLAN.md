## Phase Goal

**As a** Tankstellen-Betreiber, **I want to** in ein Freitext-Feld ein paar Sätze über meine Firma tippen und per Knopfdruck einen fertigen context.md-Entwurf bekommen, **so that** ich das context.md nicht von einer leeren Vorlage aus selbst schreiben muss — ich passe nur noch Details an.

<objective>
UI-03 (LLM-Context-Assistent) ende-zu-ende funktional. Nach diesem Plan hat der Betreiber einen 1-Klick-Weg zum ersten context.md-Entwurf.

Purpose: `context.md` schreiben aus dem Nichts ist die höchste Hürde beim Erst-Setup — der Betreiber weiß nicht welche Sektionen es geben muss, welcher Ton passt, wie formell/informell. Der LLM-Seed liefert ein passendes Grundgerüst in Sekunden.

</objective>

  <assumption>
    **W3 Trade-off — Prompts im Image, nicht als Bind-Mount:**

    Prompt-Templates für die WebUI (`webui/prompts/context-seed.txt`) werden via Dockerfile `COPY prompts/ ./prompts/` ins Image kopiert und liegen im Runtime-Container unter `/app/prompts/context-seed.txt`. Es gibt bewusst KEINEN Bind-Mount `./webui-prompts:/app/prompts:ro` in docker-compose.yml (04.01 Task 3).

    Trade-off:
    - **Vorteil:** Weniger Runtime-Bind-Mounts → kleinere Angriffsfläche für den webui-Container, weniger Host-Filesystem-Abhängigkeiten beim Kunden-Setup, ein Container-Layer weniger zu managen.
    - **Nachteil:** Prompt-Iteration (Feintuning des Seed-Prompts) erfordert Image-Rebuild und neues Deployment-Paket. Für v1 akzeptabel, weil der context-seed-Prompt selten getuned wird (Betreiber sieht den Output und editiert selbst — der Prompt liefert nur die Struktur).
    - **Kontrast zum Agent-Prompt:** Der Agent-Prompt (`agent/prompts/classify.txt`, `generate.txt`) wird in Phase 2 gerade DESHALB als Bind-Mount ausgeliefert (D-16), weil er iterativ getuned werden muss. Der Seed-Prompt hingegen ist "einmal-generieren, dann-editiert-der-Betreiber-selbst" — kein Iterations-Bedarf.
    - **Bind-Mount in v2 möglich:** Falls sich später herausstellt, dass Prompt-Iteration nötig ist, kann in einer späteren Phase ein Bind-Mount `./webui-prompts:/app/prompts:ro` in docker-compose.yml ergänzt werden, ohne Breaking Changes im Code (llm_seed.py liest PROMPT_PATH via os.getenv, ist also flexibel).

    Diese Entscheidung ist NICHT Teil von D-27..D-44 aus CONTEXT.md, sondern Umsetzungs-Freiheit gemäß RESEARCH.md-`Claude's Discretion`-Sektion.
  </assumption>
