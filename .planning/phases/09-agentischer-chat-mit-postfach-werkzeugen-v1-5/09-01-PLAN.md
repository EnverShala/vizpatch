<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Betreiber-Browser → WebUI `/chat/{id}/send` | authentifizierte, aber untrusted Eingabe (message/history/mail_context) |
| Mail-Inhalt (IMAP) → LLM | **Kern-Risiko**: fremder, untrusted Mail-Text fließt als Tool-Ergebnis ins LLM (Prompt-Injection) |
| WebUI-Container → IMAP-Postfach | mutierender Zugriff mit Fernet-entschlüsselten per-Agent-Creds |
| WebUI → Anthropic-API | Betreiber-Nachricht + Kontext + Tool-Ergebnisse verlassen den Server |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering/Elevation | Mail-Inhalt verleitet LLM zu ungefragten Tool-Aufrufen | mitigate | Tool-Ergebnisse via `wrap_tool_result` als untrusted DATEN mit Injection-Anker markiert (D-78); System-Prompt: Werkzeuge nur auf Betreiber-Wunsch; in diesem Plan gibt es nur read-only-Tools (mails_suchen), destruktive kommen erst in 09-04 mit Bestätigungs-Gate |
| T-09-02 | Info Disclosure | IBAN/Kreditkarte aus gesuchter Mail geht ans LLM | mitigate | `pii.redact` läuft in `mails_suchen` VOR Rückgabe; Test mit eingeschleuster IBAN |
| T-09-03 | Info Disclosure | api_key/IMAP-Passwort im Log oder SSE-Stream | mitigate | Secrets nie in logger/Events; Quelltext-Scan-Test `logger.*api_key == 0` |
| T-09-04 | Denial of Service | LLM ruft endlos Tools auf (Kosten/Hänger) | mitigate | `MAX_TOOL_ROUNDS`-Kappung + `CHAT_MAX_TOKENS`-Deckel + `CHAT_RATE_LIMIT_PER_MIN` (D-60, aus Phase 7 aktiv) |
| T-09-05 | Denial of Service | hängende IMAP-Verbindung blockiert Request | mitigate | `_IMAP_TIMEOUT_SECONDS` an jeder Mailbox; Handler fangen IMAP-Fehler zu `fehler`-dict |
| T-09-06 | Spoofing/Elevation | Nicht-anthropic-Provider crasht statt Fallback | mitigate | Fallback-Zweig auf beratenden `chat.stream_chat` (D-72); Test mit provider=openai |
| T-09-SC | Tampering | keine neuen Package-Installs in diesem Plan | accept | anthropic/imap-tools/cryptography bereits in pyproject (Phase 1/5); kein neuer Install → Package-Legitimacy-Gate nicht ausgelöst |
</threat_model>
