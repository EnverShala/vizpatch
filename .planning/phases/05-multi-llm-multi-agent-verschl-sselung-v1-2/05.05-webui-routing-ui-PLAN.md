<objective>
Verdrahtet die Multi-Agent-Fähigkeit in der WebUI: alle Konfig-/Status-/Steuer-Routen bekommen
einen `agent_id`-Parameter (inkl. context.md-Lesen/Schreiben pro Agent), es gibt neue CRUD-Routen
(`POST /agents`, Rename via `agents_io.rename_agent`, Delete mit Zwei-Stufen-Bestätigung), das
Formular erhält ein Agent-Dropdown (leer = Anlege-Modus, D-50) und ein LLM-Provider-Fieldset
(Anthropic|OpenAI|Google + generisches `LLM_API_KEY` + provider-abhängiger AVV-Hinweis), die
Status-Kachel wird zur Liste pro Agent (explizite `agent_statuses`-Konstruktion), und der
Zero-Reset wird auf das Multi-Agent-Modell umgestellt (alle Agenten + Fernet-Key, SEC-03).
`state_reader.py` wird per-Agent parametrisiert, `config_io.py` auf WebUI-globale Verantwortung
(Root-`.env`) bereinigt.

Purpose: MA-02 (Dropdown/CRUD), MA-04 (Status/Buttons pro Agent), LLM-01 (Provider-Dropdown-UI +
LLM_API_KEY-Feld), LLM-04 (AVV-Hinweistext), SEC-03 (Reset löscht Key mit).
Output: umgebaute `main.py`/`state_reader.py`/`config_io.py`, erweiterte Templates, aktualisierte Endpoint-Tests.
</objective>
