<objective>
Verdrahtet die Multi-Agent-Fähigkeit in der WebUI nach dem neuen D-46 (Ein-Container-Modell):
alle Konfig-/Status-Routen bekommen einen `agent_id`-Parameter (inkl. context.md pro Agent), es
gibt neue CRUD-Routen (`POST /agents`, Rename via `agents_io.rename_agent`, Delete mit Zwei-Stufen-
Bestätigung — alles OHNE Docker), Start/Stop je Agent toggelt nur das Aktiv-Flag
(`agents_io.set_agent_enabled`, wirkt ab dem nächsten Poll-Zyklus). Das Formular erhält ein
Agent-Dropdown (leer = Anlege-Modus, D-50) und ein LLM-Provider-Fieldset (Anthropic|OpenAI|Google +
generisches `LLM_API_KEY` + provider-abhängiger AVV-Hinweis). Der Status-Bereich wird zur
Übersichts-Liste ALLER Agenten (Läuft/Gestoppt aus Flag + `last_cycle`-Heartbeat, letzter Poll,
Start/Stop-Button je Zeile) plus einer globalen Container-Kachel (Phase-4-Docker-Buttons als
Admin-Funktion, `docker_ctrl` bleibt unverändert Phase-4-Stand). Der Zero-Reset wird auf das
Multi-Agent-Modell umgestellt (alle Agenten + Fernet-Key + Container-Stop, SEC-03).
`state_reader.py` wird per-Agent parametrisiert, `config_io.py` auf WebUI-globale Verantwortung
(Root-`.env`) bereinigt.

Purpose: MA-02 (Dropdown/CRUD + Einrichtungs-Flow), MA-04 (Status-Übersicht + Flag-Buttons),
LLM-01 (Provider-Dropdown-UI + LLM_API_KEY-Feld), LLM-04 (AVV-Hinweistext), SEC-03 (Reset löscht Key).
Output: umgebaute `main.py`/`state_reader.py`/`config_io.py`, erweiterte Templates, aktualisierte Endpoint-Tests.
