<objective>
Führt den internen LLM-Adapter im Agent ein und stellt Klassifikation + Draft-Generierung von
direkten Anthropic-Aufrufen auf den Adapter um. Der Adapter routet einen Single-Prompt-Aufruf
zu einem von drei SDKs (`anthropic`, `openai`, `google-genai`) mit hart verdrahteten, aber per
`.env` überschreibbaren Modell-Defaults pro Provider. Gleichzeitig wird `config.py` auf das
generische `LLM_API_KEY`/`LLM_PROVIDER`-Schema umgestellt und entschlüsselt Fernet-Secrets beim
Load (SEC-02, Agent-Seite) mit fail-fast-Verhalten bei ungültigem Key. Die Umstellung von
classify und generate erfolgt in zwei getrennten Tasks (Kontext-Entlastung).

Architektur-Kontext (neues D-46, revidiert 2026-07-16): Es gibt EINEN Agent-Container mit
Multi-Account-Loop. Dieser Plan arbeitet noch auf der bestehenden Single-Config-Struktur
(`load_config`) — Plan 05.02 (Wave 3) generalisiert danach auf `load_agent_config(agent_dir)`
pro Agent und ersetzt die hiesige Wait-Loop-Härtung durch per-Agent-Fehler-Isolation. Der
Adapter selbst ist davon unberührt: `llm_call` wird pro Agent-Durchlauf mit dessen
provider/api_key aus dem jeweiligen Config-Objekt aufgerufen.

Purpose: LLM-01 (Agent-Seite: generische Provider-/Key-Vars), LLM-02 (Adapter), LLM-03 (Modell-
Defaults), SEC-02 (Agent entschlüsselt beim Config-Load).
Output: `agent/src/llm.py`, umgestellte `classify.py`/`generate.py`, erweiterte `config.py`/`main.py`,
angepasste Agent-Tests.
</objective>
